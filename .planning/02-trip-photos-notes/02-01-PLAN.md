**Edit A — App(): add meta state + fetch useEffect**

After the existing `const [activeTripId, setActiveTripId] = useState(null);` line, add:

```javascript
const [meta, setMeta] = useState(null);

useEffect(() => {
  if (!activeTripId) {
    setMeta(null);
    return;
  }
  let cancelled = false;
  fetch(`data/${activeTripId}/meta.json`)
    .then(r => {
      if (!r.ok) return null;
      return r.json();
    })
    .then(data => {
      if (!cancelled) setMeta(data || null);
    })
    .catch(() => {
      if (!cancelled) setMeta(null);
    });
  return () => { cancelled = true; };
}, [activeTripId]);
```

Dependency array is exactly `[activeTripId]` — a string. The `cancelled` flag is a closure variable, NOT useState. The fetch path is a relative string: `data/${activeTripId}/meta.json` (no leading slash — relative to document location).

**Add PhotoItem component (insert before the PhotoCarousel function):**

```javascript
function PhotoItem({ trip, photo, index, isActive, onClick }) {
  const [errored, setErrored] = React.useState(false);
  return (
    <div className={`ta-photo ${isActive ? 'is-active' : ''}`} onClick={onClick}>
      <div className="ta-photo-img">
        {errored
          ? <svg viewBox="0 0 200 140" preserveAspectRatio="none" width="100%" height="100%">
              <rect width="200" height="140" fill="var(--tan)" />
            </svg>
          : <img
              src={`media/${trip.id}/${photo.filename}`}
              alt={photo.caption || ''}
              style={{ width: '100%', height: '100%', objectFit: 'cover', display: 'block' }}
              onError={() => setErrored(true)}
            />
        }
      </div>
      <div className="ta-photo-caption">
        <span className="ta-photo-cap">{photo.caption}</span>
      </div>
    </div>
  );
}
```

Note: `React.useState` is used (not destructured at the top of file) because PhotoItem is a new component — the destructured `useState` from line 4 of app.jsx is already available as a module-level const, so use `useState` directly (not `React.useState`). Check line 4 of app.jsx: `const { useState, useEffect, useMemo, useRef, useCallback } = React;` — yes, `useState` is available. Use `useState(false)` not `React.useState(false)`.

```javascript
function PhotoCarousel({ trip, meta }) {
  const [idx, setIdx] = useState(0);

  useEffect(() => { setIdx(0); }, [trip.id]);

  const placeholderPhotos = useMemo(() => {
    const captions = [
      'first morning on the trail',
      'pass crossing — clouds rolling in',
      'refuge dinner, soup and bread',
      'wildflowers above tree line',
      'river crossing, boots off',
      'the col at dawn',
      'janna scrambling',
      'ben at the cairn',
      'last village before the descent',
      'we made it'
    ];
    return Array.from({ length: 10 }, (_, i) => ({
      id: `${trip.id}-p${i}`,
      stage: Math.min(trip.stages.length, Math.floor(i / 10 * trip.stages.length) + 1),
      caption: captions[i],
      hue: (i * 47 + trip.year) % 60 + 20
    }));
  }, [trip]);

  const hasRealPhotos = meta && meta.photos && meta.photos.length > 0;
  const photos = hasRealPhotos ? meta.photos : placeholderPhotos;

  const visible = 5;
  return (
    <div className="ta-carousel">
      <div className="ta-carousel-head">
        <div>
          <div className="ta-carousel-overline">Field notes</div>
          <div className="ta-carousel-title">
            {hasRealPhotos
              ? `${photos.length} photos`
              : `${photos.length} photos · drop real images into media/${trip.id}/ to replace`
            }
          </div>
        </div>
        <div className="ta-carousel-controls">
          <button onClick={() => setIdx(Math.max(0, idx - 1))} disabled={idx === 0} aria-label="Previous">←</button>
          <span className="ta-carousel-counter">{idx + 1} / {photos.length}</span>
          <button onClick={() => setIdx(Math.min(photos.length - 1, idx + 1))} disabled={idx >= photos.length - 1} aria-label="Next">→</button>
        </div>
      </div>
      <div className="ta-carousel-strip">
        <div className="ta-carousel-track" style={{ transform: `translateX(calc(${-idx * (100 / visible)}%))` }}>
          {hasRealPhotos
            ? photos.map((photo, i) =>
                <PhotoItem
                  key={photo.filename}
                  trip={trip}
                  photo={photo}
                  index={i}
                  isActive={i === idx}
                  onClick={() => setIdx(i)}
                />
              )
            : photos.map((p, i) =>
                <div key={p.id} className={`ta-photo ${i === idx ? 'is-active' : ''}`} onClick={() => setIdx(i)}>
                  <PhotoPlaceholder trip={trip} photo={p} index={i} />
                  <div className="ta-photo-caption">
                    <span className="ta-photo-stage">stage {p.stage}</span>
                    <span className="ta-photo-cap">{p.caption}</span>
                  </div>
                </div>
              )
          }
        </div>
      </div>
    </div>
  );
}
```

Key points:
- `useEffect(() => { setIdx(0); }, [trip.id])` resets the carousel index on trip change — prevents out-of-range index when switching from a trip with more photos to one with fewer
- `placeholderPhotos` is in useMemo keyed to `trip` (same as original) — no change in placeholder behaviour
- `hasRealPhotos` gate: falsy meta, missing photos, or empty photos array all fall through to placeholder mode
- PhotoItem renders a real `<img>` with per-slot errored state; on load error it shows a simple tan rectangle fallback (not the full PhotoPlaceholder SVG — simpler and avoids complex pattern IDs)
- Placeholder branch is identical to the original render — no regression
- No `.ta-photo-stage` label and no `.ta-photo-tag` overlay on real photos (per UI-SPEC 2.1)
- Caption for real photos uses only `.ta-photo-cap` span (no stage prefix)
- Subtitle: real photos → `{N} photos`; placeholder → existing prompt copy
