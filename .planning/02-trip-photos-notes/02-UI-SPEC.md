## Interaction Contracts

### 2.1 Carousel — Real Image Loading (MEDIA-01, MEDIA-02)

**Trigger:** User selects a trip that has a meta.json with a non-empty `photos` array.

**Container:** `.ta-photo-img` — existing class. Dimensions: 100% width, `aspect-ratio: 4/3`, `overflow: hidden`, `background: var(--tan)`.

**Image element:**
```
<img
  src="media/{trip-id}/{filename}"
  alt="{caption text from meta.json}"
  style="width:100%; height:100%; object-fit:cover; display:block;"
/>
```
- `object-fit: cover` — fills the 4:3 frame regardless of source aspect ratio
- `alt` text = the per-photo caption string (same text shown below the thumbnail)
- The `.ta-photo-tag` overlay (filename watermark) is REMOVED when a real image is present

**While loading:** The `background: var(--tan)` of `.ta-photo-img` acts as the loading placeholder — no spinner, no skeleton. This is consistent with the existing scrapbook aesthetic.

**On image load error (broken src):** Fall back to the `PhotoPlaceholder` SVG art for that individual photo slot. Do not break adjacent photos.

**Caption binding:** `.ta-photo-caption` structure is unchanged:
```jsx
<div className="ta-photo-caption">
  <span className="ta-photo-stage">{caption}</span>
</div>
```
The `.ta-photo-stage` label ("stage N") is removed for real photos — captions from meta.json are free-form text, not stage-keyed. Use only `.ta-photo-cap`:
```jsx
<div className="ta-photo-caption">
  <span className="ta-photo-cap">{photo.caption}</span>
</div>
```

**Counter and navigation:** Counter format stays `{idx + 1} / {photos.length}`. If meta.json has fewer than 5 photos, the track still shows all photos — no minimum photo count enforced.

**Carousel subtitle line** (`.ta-carousel-title`): When real photos are loaded, show:
`{N} photos`
Not the current placeholder "drop real images into media/{trip.id}/…" string.

### 2.2 Carousel — Placeholder Fallback (MEDIA-03)

**Trigger:** Trip has no meta.json, or meta.json exists but `photos` array is absent or empty.

**Behavior:** Render `PhotoPlaceholder` SVG art exactly as it does today — no changes to the placeholder component. The 10 generated placeholder entries remain with their existing fake captions.

**Subtitle line:** Keep current text: `{N} photos · drop real images into media/{trip.id}/ to replace`

### 2.3 Stats Panel — Notes from meta.json (DATA-02, DATA-03)

**Container:** `<p className="ta-stats-blurb">` — existing class. Style: Nunito 0.95rem, line-height 1.55, color `--green-deep`, border-bottom `1px solid var(--rule)`, padding-bottom 14px.

**When meta.json notes exist:** Render `meta.notes` string directly in `<p className="ta-stats-blurb">`. Replace `{trip.blurb}` binding.

**When meta.json has no notes / no meta.json:** Fall back to `{trip.blurb}` from data.js — identical rendering, different data source.

**Text treatment:** Plain text only. No markdown parsing, no HTML injection. Newlines in the notes string are collapsed (standard browser behavior). Notes are expected to be 1–3 sentences; no truncation or overflow handling needed.

**Loading state for meta.json fetch:** The stats panel renders immediately with the data.js blurb. If an async meta.json fetch completes and notes differ, the blurb re-renders with the new text. No loading indicator in the stats panel — instant render from data.js, quiet update on fetch completion.

### 2.4 Title Override (DATA-01 optional field)

If meta.json contains an optional `title` field, it overrides the trip name in:
- `.ta-stats-title` (stats panel heading)
- `.ta-brand-sub-em` (header subtitle when trip is active)

The `trip.name` from data.js is used as the fallback if no title override exists. No visual difference — same CSS classes, different string source.

### 2.5 meta.json Fetch Pattern

**Method:** `fetch('data/{trip-id}/meta.json')` — relative path, no auth, no error modals.

**On 404 (no meta.json):** Silently swallow the error. Fall back to data.js values. No console error shown to the user.

**On parse error (malformed JSON):** Silently swallow. Fall back to data.js values.

**Timing:** Fetch is triggered when a trip is selected (`activeTripId` changes). The UI renders immediately with data.js content. meta.json data replaces it on arrival.

**Race condition:** If the user selects a different trip before fetch resolves, discard the stale response. Guard with trip ID comparison at resolution time.
