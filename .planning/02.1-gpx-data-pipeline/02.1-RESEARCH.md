## Summary

Phase 2.1 replaces the hardcoded coordinate arrays in `data.js` with a Python build script that reads GPX files referenced in per-trip `meta.json` files and regenerates `data.js`. The output must be functionally identical to the current file тАФ same IIFE structure, same field names, same coordinate format, same runtime stats computation.

The data folder currently uses geographic names (`brittany/`, `corsica/`, etc.) while the app uses short trip IDs (`bia`, `rab`, etc.). These are separate namespaces. The build script must target the trip-ID folders (`data/<trip-id>/`) where `meta.json` lives, and `gpx_stages` paths will be relative to that folder or reference the geographic subfolders.

The GPX files are Komoot-exported GPX 1.1 format with a default XML namespace (`xmlns="http://www.topografix.com/GPX/1/1"`), which requires namespace-aware parsing in Python. Each GPX contains `<trk>/<trkseg>/<trkpt lat lon><ele>` structure. The build script must decimate trackpoints to 70 per stage to keep the output file size comparable to the current `data.js`.

**Primary recommendation:** Write `scripts/build_data.py` using Python's `xml.etree.ElementTree` with namespace registration. Decimate to 70 points per stage using uniform skip (`step = max(1, len(points) // 70)`). The script reads all `data/<trip-id>/meta.json` files, extracts the existing static fields (id, name, year, country, hub, blurb, months, color), parses the GPX files listed in `gpx_stages`, and writes a new `data.js` with an identical IIFE structure.

## GPX File Format

**Format:** GPX 1.1, Komoot export [VERIFIED: reading GPX file headers]

```xml
<?xml version='1.0' encoding='UTF-8'?>
<gpx version="1.1" creator="https://www.komoot.de"
     xmlns="http://www.topografix.com/GPX/1/1" ...>
  <metadata>
    <name>Track Name</name>
  </metadata>
  <wpt lat="..." lon="...">...</wpt>   <!-- optional waypoints -->
  <trk>
    <name>Track Name</name>
    <trkseg>
      <trkpt lat="48.315274" lon="-4.548743">
        <ele>17.083327</ele>
        <time>...</time>
      </trkpt>
      <!-- hundreds or thousands of trkpt elements -->
    </trkseg>
  </trk>
</gpx>
```

**Critical:** The `xmlns="http://www.topografix.com/GPX/1/1"` default namespace means `xml.etree.ElementTree` requires namespace-qualified tag searches. [VERIFIED: reading GPX files + Python xml.etree behavior]

```python
ns = {'gpx': 'http://www.topografix.com/GPX/1/1'}
trkpts = root.findall('.//gpx:trkpt', ns)
```

**Encoding:** UTF-8 (declared in XML header). Some GPX filenames contain non-ASCII characters (e.g., `Stur├н─Н`, `Wanderung`). Python `xml.etree.ElementTree.parse()` with `parser=ET.XMLParser(encoding='utf-8')` handles this reliably. [VERIFIED: encoding error without explicit parser; success with explicit parser]

**Coordinate precision:** `lat`/`lon` attributes are floats with 6 decimal places. `<ele>` is a float element (e.g., `17.083327`). The output format in data.js truncates to 5 decimal places for lat/lon and rounds elevation to integer. [VERIFIED: comparing GPX values to data.js values]

## Decimation: 70 Points Per Stage

**Finding:** Every single stage in the current data.js contains exactly 70 coordinate points. [VERIFIED: counting all 29 stages across 7 trips]

| Trip | Stages | Total pts | Pts/stage |
|------|--------|-----------|-----------|
| bia | 6 | 420 | 70 |
| rab | 3 | 210 | 70 |
| oro | 2 | 140 | 70 |
| sar | 7 | 490 | 70 |
| slo | 5 | 350 | 70 |
| swi | 5 | 350 | 70 |
| rue | 5 | 350 | 70 |

**Decimation ratio varies** by trip (1:4 to 1:20 depending on GPX density): [VERIFIED: counting GPX trkpts vs data.js points]

| Trip | GPX trkpts | data.js pts | Ratio |
|------|-----------|-------------|-------|
| bia | 6026 | 420 | 1:14 |
| rab | 1460 | 210 | 1:7 |
| oro | 2754 | 140 | 1:20 |
| sar | 5764 | 490 | 1:12 |
| slo | 1379 | 350 | 1:4 |
| swi | 3682 | 350 | 1:11 |
| rue | 1826 | 350 | 1:5 |

**Decimation algorithm:** Uniform index-based skip. Formula: `step = max(1, len(points) // 70)`. Take every `step`-th point. This produces between 70 and `70 + step - 1` points per stage; always include the first and last point to preserve track endpoints. [ASSUMED тАФ uniform skip inferred from index pattern analysis; Ramer-Douglas-Peucker not ruled out but uniform skip reproduces the observed ~70-pt-per-stage pattern without external dependencies]

## Build Script: Implementation Pattern

**Python 3 availability:** [VERIFIED: `python --version` тЖТ Python 3.9.0]
**xml.etree.ElementTree:** [VERIFIED: stdlib, no install needed]
**Script location:** `scripts/build_data.py` (per GPX-02)

**Algorithm:**

```python
#!/usr/bin/env python
"""Build script: reads data/<trip-id>/meta.json files, parses GPX, writes data.js"""

import json
import os
import re
import xml.etree.ElementTree as ET

GPX_NS = {'gpx': 'http://www.topografix.com/GPX/1/1'}
TARGET_POINTS = 70
DATA_DIR = os.path.join(os.path.dirname(__file__), '..', 'data')
OUTPUT_PATH = os.path.join(os.path.dirname(__file__), '..', 'data.js')

def parse_gpx_stage(gpx_path):
    """Parse one GPX file, return list of [lat, lon, ele_int] decimated to ~70 pts."""
    tree = ET.parse(gpx_path, parser=ET.XMLParser(encoding='utf-8'))
    root = tree.getroot()
    trkpts = root.findall('.//gpx:trkpt', GPX_NS)
    if not trkpts:
        raise ValueError(f'No trkpt elements in {gpx_path}')
    # Collect all points
    points = []
    for pt in trkpts:
        lat = round(float(pt.get('lat')), 5)
        lon = round(float(pt.get('lon')), 5)
        ele_el = pt.find('gpx:ele', GPX_NS)
        ele = round(float(ele_el.text)) if ele_el is not None else 0
        points.append([lat, lon, ele])
    # Uniform skip decimation to TARGET_POINTS
    step = max(1, len(points) // TARGET_POINTS)
    decimated = points[::step]
    # Always include last point
    if decimated[-1] != points[-1]:
        decimated.append(points[-1])
    return decimated

def format_stage(points):
    """Format stage as compact JS array string."""
    inner = ', '.join(f'[{p[0]},{p[1]},{p[2]}]' for p in points)
    return f'[{inner}]'

def build():
    trips = []
    for trip_id in sorted(os.listdir(DATA_DIR)):
        meta_path = os.path.join(DATA_DIR, trip_id, 'meta.json')
        if not os.path.isfile(meta_path):
            continue
        with open(meta_path, 'r', encoding='utf-8') as f:
            meta = json.load(f)
        if 'gpx_stages' not in meta:
            continue  # skip trips not yet configured
        # Parse each GPX
        meta_dir = os.path.dirname(meta_path)
        stages = []
        for gpx_rel in meta['gpx_stages']:
            gpx_path = os.path.normpath(os.path.join(meta_dir, gpx_rel))
            stages.append(parse_gpx_stage(gpx_path))
        trips.append({'meta': meta, 'stages': stages})
    # Write data.js
    write_data_js(trips)

# ... write_data_js emits the IIFE
```

**Key output requirement:** The `data.js` IIFE, haversine function, and runtime stats computation block are static boilerplate тАФ copy verbatim from current `data.js` lines 123тАУ160. Only the `TRIPS` array content changes.

## Common Pitfalls

### Pitfall 1: Default Namespace in GPX XML
**What goes wrong:** `root.findall('.//trkpt')` returns empty list.
**Why it happens:** GPX 1.1 files declare a default namespace `xmlns="http://www.topografix.com/GPX/1/1"`. ElementTree requires namespace-qualified selectors.
**How to avoid:** Always pass `ns = {'gpx': 'http://www.topografix.com/GPX/1/1'}` and use `root.findall('.//gpx:trkpt', ns)`.
**Warning signs:** Empty stages list with no error.

### Pitfall 2: Windows Path Separators
**What goes wrong:** `os.path.join` produces backslash paths; GPX path resolution fails.
**Why it happens:** Running on Windows 11.
**How to avoid:** Use `os.path.normpath(os.path.join(meta_dir, gpx_rel))` тАФ normpath handles mixed separators.

### Pitfall 3: Encoding on Windows
**What goes wrong:** `ET.parse(path)` raises `UnicodeEncodeError` for GPX files with non-ASCII filenames or content.
**Why it happens:** Default Windows codec (cp1252) cannot encode characters like `─Н`, `├н`.
**How to avoid:** Always pass `parser=ET.XMLParser(encoding='utf-8')` to `ET.parse()`.

### Pitfall 4: Sardinia Duplicate and Approach File
**What goes wrong:** Script picks up all 9 sardinia GPX files; Stage 4 appears twice; approach walk adds wrong route.
**Why it happens:** sardinia/ has a `(1)` variant of Stage 4 and a pre-hike file.
**How to avoid:** `gpx_stages` in `data/sar/meta.json` explicitly lists the 7 correct files; the script only reads what's listed, not all files in the folder.

### Pitfall 5: Coordinate Precision Drift
**What goes wrong:** Coordinates rounded differently from original, causing visual drift or stat discrepancy.
**Why it happens:** GPX has 6 decimal places; original data.js has 5.
**How to avoid:** `round(float(lat), 5)` for lat/lon; `round(float(ele))` for elevation (integer).

### Pitfall 6: Stage Order in meta.json
**What goes wrong:** Stages appear in wrong order on the map; track pins numbered incorrectly.
**Why it happens:** `os.listdir()` or `glob` does not guarantee alphabetical order.
**How to avoid:** `gpx_stages` in meta.json is an ordered array тАФ the script processes them in list order. Do NOT sort GPX files automatically; respect the user's declared order.

### Pitfall 7: Missing Trip ID Folders
**What goes wrong:** Only `bia` has a trip-ID folder; other 6 trips have no `data/<trip-id>/` folder or meta.json.
**Why it happens:** Prior phases only scaffolded `bia`.
**How to avoid:** Phase plan must include task to create `data/rab/`, `data/oro/`, etc. with meta.json for each.

```python
# Source: exact copy of data.js lines 123-160
IIFE_RUNTIME = '''
  var R = 6371;
  function haversine(a, b) {
    var phi1=a[0]*Math.PI/180, phi2=b[0]*Math.PI/180;
    var dphi=(b[0]-a[0])*Math.PI/180, dlam=(b[1]-a[1])*Math.PI/180;
    var x=Math.sin(dphi/2)*Math.sin(dphi/2)+Math.cos(phi1)*Math.cos(phi2)*Math.sin(dlam/2)*Math.sin(dlam/2);
    return 2*R*Math.asin(Math.sqrt(x));
  }

  TRIPS.forEach(function(trip) {
    var dist=0, gain=0, hi=-Infinity, lo=Infinity, all=[];
    trip.stages.forEach(function(stage) {
      for (var i=0; i<stage.length; i++) {
        var p=stage[i];
        if (p[2]>hi) hi=p[2];
        if (p[2]<lo) lo=p[2];
        all.push(p);
        if (i>0) {
          dist += haversine(stage[i-1], stage[i]);
          var d = stage[i][2]-stage[i-1][2];
          if (d>0) gain+=d;
        }
      }
    });
    trip.stats = {
      distance: Math.round(dist),
      elevationGain: Math.round(gain),
      highest: Math.round(hi),
      lowest: Math.round(lo),
      stages: trip.stages.length,
      days: trip.stages.length
    };
    trip.allPoints = all;
  });

  window.TRIPS = TRIPS;
  window.totalDistance = TRIPS.reduce(function(s,t){return s+t.stats.distance;}, 0);
  window.totalElevation = TRIPS.reduce(function(s,t){return s+t.stats.elevationGain;}, 0);
'''
```
