# Phase 2.1 Plan 02: Write build_data.py and regenerate data.js Summary

Python build script `scripts/build_data.py` reads all 7 `data/<trip-id>/meta.json` files, parses GPX stage files via `xml.etree.ElementTree` with explicit namespace, decimates each stage to ~70 points via uniform skip, and rewrites `data.js` with an identical IIFE structure — replacing all hardcoded coordinate arrays with GPX-derived data.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Removed dashArray that rendered odd stages as dots**
- **Found during:** Task 3 (human verification)
- **Issue:** `dashArray: i % 2 ? '1 6' : null` on the track polyline caused every other stage (odd-indexed) to render as a dotted line with 1px dots spaced 6px apart — visually appearing as no track on those segments
- **Fix:** Removed `dashArray` property entirely; all stages now render as solid 3.5px polylines
- **Files modified:** app.jsx
- **Commit:** f313ace

**2. [Rule 3 - Blocking] Removed TweaksPanel (broken import after tweaks-panel.jsx deletion)**
- **Found during:** Inline fix after human checkpoint
- **Issue:** tweaks-panel.jsx was deleted in a prior cleanup; app.jsx still referenced `useTweaks`, `TweaksPanel`, `TweakSection`, `TweakRadio`, `TweakText`, `TweakColor`, `TweakToggle` — the page would fail to load with a Babel transpilation error
- **Fix:** Removed TweaksPanel from app.jsx, removed the `<script src="tweaks-panel.jsx">` tag from index.html, inlined the three configurable values (title, tile theme, track colour) as constants
- **Files modified:** app.jsx, index.html
- **Files deleted:** tweaks-panel.jsx (was already unlinked at source; git deletion committed here)
- **Commit:** f313ace
