# Plan 04-01 — GitHub Pages deployment

## Goal

Publish the site at `https://blenback.github.io/urlaub` by pushing the current main branch to GitHub and enabling Pages from the repository settings.

## Path audit (pre-flight)

All asset references are relative URLs — no sub-path changes needed:

| File | Reference | Resolves under /urlaub |
|------|-----------|------------------------|
| index.html | `data.js` | `/urlaub/data.js` ✓ |
| index.html | `app.jsx` (Babel XHR) | `/urlaub/app.jsx` ✓ |
| app.jsx | `data/${id}/meta.json` | `/urlaub/data/*/meta.json` ✓ |
| app.jsx | `media/${id}/${photo}` | `/urlaub/media/*` ✓ |
| app.jsx | `href="snacks.html"` | `/urlaub/snacks.html` ✓ |
| snacks.html | `snacks.jsx` (Babel XHR) | `/urlaub/snacks.jsx` ✓ |

| snacks.jsx | `media/snacks/${photo}` | `/urlaub/media/snacks/*` ✓ |
| snacks.jsx | `href="index.html"` | `/urlaub/index.html` ✓ |

All CDN links (React, Leaflet, Babel, Google Fonts) are absolute `https://` URLs — unaffected.

**Conclusion**: No code changes required. Site will work at the `/urlaub` sub-path as-is.

## Tasks

### Task 1 — Push main branch to GitHub

```bash
git push origin main
```

If already up-to-date, this is a no-op.

### Task 2 — Enable GitHub Pages via repository API

Use the GitHub REST API (requires a GitHub personal access token with `repo` scope):

```powershell
$token = "YOUR_GITHUB_TOKEN"
$headers = @{
    Authorization = "Bearer $token"
    Accept = "application/vnd.github+json"
    "X-GitHub-Api-Version" = "2022-11-28"
}
$body = @{
    source = @{
        branch = "main"
        path = "/"
    }
} | ConvertTo-Json

Invoke-RestMethod -Uri "https://api.github.com/repos/blenback/urlaub/pages" `
    -Method POST -Headers $headers -Body $body -ContentType "application/json"
```

**Alternatively (no token needed):** Enable via GitHub UI:
1. Go to https://github.com/blenback/urlaub/settings/pages
2. Under "Source", select: **Deploy from a branch**
3. Branch: **main** / Folder: **/ (root)**
4. Click **Save**

GitHub Pages deployment takes ~1–2 minutes after enabling.

### Task 3 — Verify deployment

After 1–2 minutes, open: `https://blenback.github.io/urlaub`

Check:
- [ ] Map page loads and shows all 7 trip markers
- [ ] Clicking a marker zooms in and shows the GPX track
- [ ] "Trail Snacks →" link navigates to snacks.html
- [ ] Snacks page loads and shows the 4 sample cards sorted by joint score
