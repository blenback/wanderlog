# Plan 03-02 — Sample data + map-page nav link

## Goal

Populate `snacks.json` with 4 real-feeling sample entries so the gallery has content to display. Wire the nav link from `index.html` (map page) to `snacks.html` by editing the `Header` component in `app.jsx`.

## Context

- `snacks.json` must live at the repo root (fetched as `fetch('snacks.json')` from snacks.jsx)
- Schema: `id`, `name`, `country`, `photo`, `ben: { score, note }`, `janna: { score, note }`
- `photo` value is a filename only — served from `media/snacks/<filename>`; photos don't need to exist yet
- Navigation in index.html is driven by the `Header` component in `app.jsx` — the nav is a `<nav className="ta-nav">` containing the "← All trips" button. Add a Snacks link alongside it.
- The Snacks link must be an `<a href="snacks.html">` (not a React button) since it navigates to a different page
- The link should be styled identically to `.ta-nav-back` — use `.ta-nav-back` class or a matching `<a>` tag

### Task 2 — Edit `app.jsx` Header to add Snacks nav link

The `Header` component in `app.jsx` currently renders:

```jsx
<nav className="ta-nav">
  {activeTrip &&
    <button className="ta-nav-back" onClick={onHome}>
      <span aria-hidden>←</span> All trips
    </button>
  }
</nav>
```

Add a "Trail Snacks →" link as an `<a>` tag. It should be always visible (not gated on activeTrip). Place it after the existing button so "All trips" appears first when both are shown.

Target result:

```jsx
<nav className="ta-nav">
  {activeTrip &&
    <button className="ta-nav-back" onClick={onHome}>
      <span aria-hidden>←</span> All trips
    </button>
  }
  <a href="snacks.html" className="ta-nav-back">
    Trail Snacks <span aria-hidden>→</span>
  </a>
</nav>
```

The `.ta-nav-back` class already has the correct pill-button styling. Use `<a>` (not `<button>`) so the browser handles the page navigation.
